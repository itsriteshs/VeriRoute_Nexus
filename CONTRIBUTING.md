# Contributing

Use feature branches. Never directly push to `main` unless the team agrees it is an emergency.

## Branch Naming

- `person1/backend-routing`
- `person1/immune-checks`
- `person2/dashboard-shell`
- `person2/digital-twin`
- `person3/hardware-scan`
- `person3/slides-demo`

## Pull Request Rules

- Keep PRs focused.
- Update the matching `what_is_done_person_X.md` file before every commit.
- Mention any changed API shape in `API_CONTRACT.md`.
- Include screenshots/video for frontend or hardware-visible changes.
- Include test proof or manual verification steps.

## Commit Message Examples

- `feat(backend): add demo route graph seed data`
- `feat(frontend): scaffold trust board panel`
- `feat(hardware): add smart hub scan payload stub`
- `docs(api): update hardware scan contract`
