# Naming And Positioning

Use these names consistently in README, deck, UI labels, and judge narration.

| Name | Definition | Judge-friendly sentence |
| --- | --- | --- |
| PacketFlow Routing Engine | Dynamic next-hop protocol for parcel movement over a hub graph. | "PacketFlow chooses the next trusted hop instead of following a brittle fixed route." |
| ImmuneNet Anomaly Engine | Movement-claim immune system that blocks suspicious logistics events. | "ImmuneNet treats fake scans like infections in the network." |
| Proof-of-Movement Ledger | Hash-linked event record of accepted, blocked, and rerouted movement claims. | "Every movement claim leaves an auditable proof trail." |
| SwarmFlow Digital Twin | Live dashboard of hubs, parcels, route decisions, alerts, and trust state. | "SwarmFlow lets judges watch the logistics network react in real time." |
| AgentOps Disruption Replanner | Scenario engine that responds to failed, overloaded, risky, or unsafe route conditions. | "AgentOps replans when the network environment changes." |
| SmartHub Relay Node | ESP32/RFID/QR/GPS/temperature edge device concept for physical handoff proof. | "A SmartHub turns a scan into a verified physical event." |
| Edge Trust Handshake | ESP-NOW/BLE-style trust exchange between nearby hubs and parcel tags. | "The edge devices share local proof before the backend trusts a handoff." |
| Cold-Chain Immune Check | Temperature-aware scan and routing check for sensitive parcels. | "Medicine does not just need location; it needs condition proof." |
| Trust Decay Model | Hub reputation update after accepted, warning, blocked, rerouted, or tamper events. | "Untrustworthy hubs become less attractive to PacketFlow." |
| Movement Claim Validation | The full Proof-of-Movement process for scanner identity, geofence, speed, graph, temperature, tamper, and optional hardware signals. | "A scan is not accepted until the movement claim is plausible." |

## Positioning Rules

- Lead with **PacketFlow ImmuneNet** as the product name.
- Use **VeriRoute Nexus** only as the repository/original team codename if needed.
- Do not call it a parcel tracking app.
- Do not imply production-grade live hardware deployment unless the physical demo is shown.
- Say "scenario simulator" or "digital twin dashboard" instead of "full physics simulator."
- Say "hardware-ready" when referring to firmware/CAD/PCB/simulator without a connected device in the judge environment.
